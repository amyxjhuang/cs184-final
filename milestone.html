Parallelized Bezier Curve Font Rendering

 

Due Tuesday, April 26th, 11:59pm.

Group: Vincent Chiang, Alek Kozminski, Anderson Lam, Amy Huang

 

In this project, we will implement a Bezier-Curve based font renderer. We will make this renderer fast by parallelizing it on the CPU through use of the OpenMP library, and on the GPU through use of CUDA.

 

Progress Summary / Preliminary Results

 

So far, we have implemented code that reads .ttf files and outputs the points to a .bzc file, in the same format as those used in project 2. After doing this, we are able to plot the points and generate curves from the project 2 code that we are using as our foundation. However, the curves that are generated are not correct and what we want. We are currently debugging this issue and looking into how the project 2 code reads the .bzc code to better understand how we can modify it for our purposes.

We are loading in the .ttf files not through a package in C++ as we found it hard to find a suitable and easy-to-use one. Instead, our alternative solution is to load the .ttf files through a package (freetype) in Python. We created a simple python executable to take in arguments of font file name, size, and character, which outputs the number of points and all the coordinates (same format as the curve.bzc files). From those points we generated the above images.

 

In the meantime, we have split our work. Some of us are looking into the project 2 code base and trying to figure out how it works so we can modify it appropriately. Others are working off a python notebook to figure out how we can correct the fully evaluated bezier curve. Since we are much more familiar with Python, it will be easier and faster for us to figure out our issues. To do this, we have built a bezier curve plotter in Python using numpy, scipy, and matplotlib.

Encountered Issues / Progress Reflection

 

We spent a lot of time trying to figure out how to read the ttf files since the representation is kind of convoluted- as there are lots of included points that are not supposed to be rendered but are needed to create the Bezier curve. We really strived to keep the code in C++ by finding a package that could read fonts in C++ and give us the desired points, but we had a lot of trouble. Also, our code would work on some fonts and not others. However, we were able to try to find more information by reading Stack Overflow and documentation. Going forward we have a clearer understanding of what to do.

 

Aside from issues just reading the font files, we are having more issues than we expected for rendering the final bezier curve. We thought that it would be more seamless, but there is a lot more research and tinkering that needs to be done.

 

In addition, we were overly optimistic about our timeline. We had expected to be done with our CPU implementation by now and also try to speed it up during this week. All of our schedules were busier than expected due to exams (had not accounted for the midterm last week) and other classwork. Our schedule now is much more realistic and we believe that after figuring out the issues we have now, GPU implementation should be easier.
