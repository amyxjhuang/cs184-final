CS184 Final Proposal
Parallelized Bezier Curve Font Rendering  

Due Monday, April 11th, 11:59pm.

Group: Vincent Chiang, Alek Kozminski, Anderson Lam, Amy Huang

In this project, we will implement a Bezier Curve based font renderer. We will make this renderer fast by parallelizing it on the CPU through use of the OpenMP library, and on the GPU through use of CUDA.

Problem Description

Font rendering is a problem and here’s why. There are many ways to render fonts and with this, there are many tradeoffs (aliasing, not scalable, etc) when choosing a font rendering technique. With Bezier curve font rendering it allows us to efficiently represent characters that can be anti-aliased and scaled using curves that represent the outlines. In addition we will parallelize this in order to make it even faster than traditional font rendering. This task is important because it is used frequently in our high tech world and serves as building more knowledge. This will be challenging for us as this problem requires us to gather a deeper understanding of how font files work (finding the right packages to help us), how bezier curves work, and how to render it all without the guidance of course staff. We will solve this problem by understanding how font files can be loaded then find ways to draw Bezier curves from that information. From there, we aim to figure out how to speed up rendering with CUDA.

Goals and Deliverables

We intend to deliver code for a parallelized font renderer, as well as a sample of a font rendered by it, and compare rendering times between it and FreeType for large samples of text. We will start by creating parallel code for a CPU using OpenMP, and will then attempt to port this to a GPU-based implementation with CUDA.

We will render these using the CPUs and GPUs provided to us on Hive.

For our deliverables, we will explain our implementation and detail the process we went through concisely. This will include loading TrueType files, writing a Bezier curve rasterizer, and displaying the fonts. Images that we will show will at least include individual letters, words, and long sentences in varying fonts based on the different font files.

We plan to achieve a complete and working system where you can load any .ttf file, provide the text you want to render, then display it from CPU rendering. We expect our system to not be too slow and flexible with different types of fonts and text. For us, this is our baseline project and would mean we have a successful project. Being able to take what we learned in class by rendering simple points to then rendering fonts appropriately will be challenging. Unexpected delays would be from having a much harder time than expected to find methods to read in the font files, use that data properly, and then render (we may not be able to use the previous rendering screen from the Bezier curve project).

If we have extra bandwidth, we hope to make our display of text more interactive (drag points like in our class project) or informative and utilize CUDA for GPU rendering. This is a stretch goal as we have not used CUDA before and have not used GPU rendering in class. Once we speed up our program, we will generate graphs to compare the different run times. We could also potentially generate fonts based on an interactive window to modify the Bezier curves.

In order to assess our system’s quality, we want to ensure our Bezier curve rasterizer is accurate, which would require human judgment/testing. We also want our CPU rendering to have a decent runtime, where we should be able to see better performance when rendering with GPUs. Based on initial research, we expect at least a 2x performance increase.

We believe we would be able to complete this project as we have some foundational knowledge for rendering, Bezier curves, and have worked on building new implementations in the past. We are also hopeful that we will be able to gather sufficient resources.
